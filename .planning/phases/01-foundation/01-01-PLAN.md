**Step 2 — DB schema (`src/lib/db/schema.ts`):**

Use the exact schema from RESEARCH.md code examples section. Key points:
- `brands`: id UUID PK, name, url, brandContextBlock (text — compiled at save time), toneDescription, targetAudience, styleRules (jsonb `string[]`), pluginFacts (jsonb `PluginFact[]`), exampleArticle1, exampleArticle2, targetKeywords (jsonb `string[]`), createdAt, updatedAt
- `performanceData`: id UUID PK, brandId (FK → brands.id NOT NULL), pageTitle, sessions (integer), bounceRate (real nullable), formatType (text), uploadedAt
- `articleDrafts`: id UUID PK, brandId (FK → brands.id NOT NULL), topic, targetKeyword, title, metaDescription, outline (jsonb `ArticleOutline`), fullContent (text), status (text default 'draft', typed as `'generating' | 'draft_ready' | 'error'`), selectedFormat, generatedAt, updatedAt

Import types from `@/types` in the schema file.
