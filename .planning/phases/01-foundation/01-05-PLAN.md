**Generate page (`src/app/(dashboard)/generate/page.tsx`):**

Server Component. Fetches brand profile (via `getBrandProfile()`) and performance data (via `getPerformanceData(brandId)`). If no brand profile: show "Set up your brand profile first" with link to /brand.

Passes `brandContextBlock` and `topFormats` (top 3 format types by session count, derived from performance data) as props to the `<ArticleStream>` client component.

**FormatPicker component (`src/components/performance/FormatPicker.tsx`):**

Client component. Props: `topPerformingFormat: string | null` (e.g., 'how-to', derived from highest-traffic format in perf data).

Renders two choices per CONTEXT.md locked decision — user-controlled, not automatic:
1. "Use top-performing format" — shows the detected format type as a Badge (e.g., "how-to based on your performance data"). Only available if performance data has been uploaded.
2. "Choose format manually" — dropdown with options: How-To Guide, List Post, Comparison Post, General Article

Selected format is passed to the generation call. Default: "Choose format manually" with "General Article" pre-selected.

**ArticleStream component (`src/components/article/ArticleStream.tsx`):**

Client component. Manages the full generation state machine.

State:
- `phase`: 'idle' | 'outline' | 'sections' | 'complete' | 'error'
- `outline`: ArticleOutline | null
- `sections`: Record<number, string> (sectionIndex → streamed content so far)
- `currentSectionIndex`: number
- `draftId`: string | null

Props: `brandContextBlock: string`, `topPerformingFormat: string | null`

UI elements:
1. Topic input: `<textarea>` labeled "Topic or keyword to write about" (required, single line feel but multiline supported)
2. Format picker: renders `<FormatPicker>` component
3. "Generate Article" button — disabled while generating

Generation sequence (executed on button click):
1. `initArticleGeneration(brandId, topic, selectedFormat)` → get `draftId`
2. POST to `/api/generate/outline` → receive ArticleOutline JSON, set `outline`, set `phase: 'sections'`
3. Display outline structure immediately (all section headings rendered as skeleton)
4. For each body section (index 0 to sections.length - 3, assuming last 2 are intro/conclusion):
   - POST to `/api/generate/section` with `{outline, sectionIndex: i, sectionType: 'section', brandContextBlock}`
   - Stream response using the AI SDK's `readDataStream` or a custom `ReadableStream` reader
   - Append tokens to `sections[i]` state as they arrive
5. After all body sections: POST for introduction (`sectionType: 'introduction'`, `bodySections: Object.values(sections)`)
6. After introduction: POST for conclusion (`sectionType: 'conclusion'`)
7. After all complete: assemble full content string, call `updateArticleDraft(draftId, { fullContent, title: outline.title, metaDescription: outline.metaDescription, targetKeyword: outline.targetKeyword, outline, status: 'draft_ready' })`
8. Set `phase: 'complete'`

**Important:** The order is body sections → introduction → conclusion. Per RESEARCH.md, intro/conclusion are written last so they accurately summarize what was actually written (not just what the outline promised).

Display: While streaming, show the outline structure with each section's heading rendered as an H2/H3 header. As content streams into a section, it appears below its heading. Use a ref to scroll the user to the currently-streaming section.

**ArticleOutput component (`src/components/article/ArticleOutput.tsx`):**

Renders the completed article as formatted rich text. Props: `outline: ArticleOutline`, `sections: Record<number, string>`.

Renders in this order:
- H1: outline.title
- Meta description line: small text showing the meta description (for reference, not part of the copy)
- Target keyword badge
- For each section in outline.sections: render the heading as H2 or H3 (per headingLevel), then render the section content as paragraphs (split on `\n\n`)
- At the end: CTA note — "Plugin tie-in: [outline.ctaPlugin]" in a highlighted box

"Copy article" button: uses `navigator.clipboard.writeText()` to copy the full assembled HTML or markdown to clipboard. Format: headings as `#`/`##`/`###` markdown, paragraphs as plain text. This is what the user pastes into WordPress Gutenberg.

Show a toast (use a simple `alert()` or shadcn/ui Toast) confirming "Article copied to clipboard".

Per CONTEXT.md locked decision: Output is formatted rich text with headings already applied — copy-paste straight into WordPress block editor. Use semantic heading levels in the rendered output.

**RegenerateButton component (`src/components/article/RegenerateButton.tsx`):**

Simple button: "Regenerate Article". Appears after generation completes (`phase === 'complete'`). On click, resets the generation state and triggers the full pipeline again for the same topic and format. This calls `updateArticleDraft(draftId, { status: 'generating' })` first, then re-runs the pipeline.

Per CONTEXT.md locked decision: no section-level regeneration in v1 — full regeneration only.

**Drafts page (`src/app/(dashboard)/drafts/page.tsx`):**

- Generation date
- "Open" button — links to `/generate?draftId={id}` to re-display the draft
- Empty state: "No drafts yet. Go to Generate to create your first article."

Per CONTEXT.md locked decision: Articles are saved automatically — user can return, copy again, or regenerate later.
